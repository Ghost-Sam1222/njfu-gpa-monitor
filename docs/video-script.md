# NJFU GPA Monitor 云端配置教程脚本

这是一支放在 README 顶部的通用入门视频，不把 Bark 当作默认或唯一渠道。成片建议控制在 2 分钟以内，旁白使用真人或自然的 Siri 级中文声线；语速约每秒 4.6 至 5 个汉字，镜头切换间隔不超过 1 秒。

| 时间 | 画面 | 旁白 |
| --- | --- | --- |
| 00:00-00:12 | Pages 首页 | 这是南林成绩监控。它用 GitHub Actions 定时查看教务系统，发现新成绩后通知你。整个配置只用浏览器，不下载代码，也不需要终端。 |
| 00:12-00:28 | 输入 GitHub 用户名，创建仓库 | 先填自己的 GitHub 用户名，点“创建监控仓库”。这是你的个人仓库；公开代码可以分享，账号、密码和通知密钥不会写进代码。 |
| 00:28-00:43 | 打开云端设置，创建 Codespace | 回到入口，点“打开云端设置”，确认仓库后创建 Codespace。第一次通常等两到四分钟。设置页是私有端口，只有你登录 GitHub 后能打开。 |
| 00:43-00:59 | 学号、密码、学期、验证按钮 | 在这里填写学号、密码和学期，先点“验证教务登录”。信息会直接写进你自己的 GitHub Secrets，不经过公开 Pages，也不要放进截图或聊天记录。 |
| 00:59-01:17 | 通知渠道教程页 | 选择一个通知渠道即可。邮件适合所有设备；Bark 适合 iPhone 和 iPad；飞书、钉钉、企业微信、Telegram 和 Slack 适合已有群聊。不会配置时点“逐步教程”，按步骤拿到授权码或 Webhook。 |
| 01:17-01:32 | 频率与停止日期 | 选择检查频率。通常六小时或十二小时就够了。再填一个预计所有成绩公布的停止日期；默认会在那天发一份当前成绩单，避免你还要计算课程数量。 |
| 01:32-01:46 | 完成配置，测试通知 | 保持“保存后触发一次通知测试”开启，点“完成配置”。只有目标 App 或邮箱真的收到测试通知，才算部署完成。 |
| 01:46-01:58 | 通知示例与成绩单 | 新成绩通知只显示课程、分数、绩点、学分和当前平均绩点。最终邮件会多一份可排序的成绩单。 |
| 01:58-02:10 | 下学期修改学期和日期 | 下学期只需要重新打开设置页，改学期、开始日期和停止日期。平时无需开电脑；停止日期过后，工作流会在登录教务系统前结束。 |

## 录制规范

- 不展示真实学号、密码、Cookie、Webhook、邮箱授权码或 Device Key。
- Pages、Codespaces、设置页和通知教程四个画面都要录到，避免视频误导用户只能使用 Bark。
- 每次点击后用 0.4 至 0.8 秒过渡；Codespace 启动等待用加速画面和文字说明，不留静默空档。
- 结尾不要承诺“绝对不会变”。应说明：学校登录页或 GitHub 规则变动时，开启的每周公开代码同步会带来兼容修复，Secrets 不会被读取或覆盖。
