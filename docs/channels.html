选择通知渠道

只配置一种就能使用。这里不会要求你粘贴任何密钥，拿到凭据后再回到私有设置页填写。

最通用：邮件所有手机和电脑都能收，QQ、163 可用。
Apple 设备：Bark通知最快、最简洁，仅适合 iPhone 和 iPad。
已有聊天 App飞书、钉钉、企业微信、Telegram、Slack。

邮件：推荐给大多数人

本项目只通过 SMTP 发信，不读取邮箱，不需要 IMAP。授权码不是邮箱登录密码。

QQ 邮箱

  1. 在电脑浏览器登录 QQ 邮箱。
  2. 进入“设置 → 账号”，找到 POP3/IMAP/SMTP 服务。
  3. 开启 IMAP/SMTP 或 POP3/SMTP，按页面完成安全验证，生成 16 位授权码。
  4. 设置页填写发件邮箱、授权码和收件邮箱；SMTP 地址和端口留空即可自动识别。

163 / 126 邮箱

  1. 登录 163 邮箱 或 126 邮箱。
  2. 进入“设置 → POP3/SMTP/IMAP”，开启 SMTP 服务并设置客户端授权密码。
  3. “SMTP 授权码”填写该授权密码，不要填写网页登录密码；服务器和端口可留空。

Bark：iPhone / iPad

  1. 从 App Store 安装 Bark 并打开。
  2. 在 App 首页复制测试地址，格式类似 https://api.day.app/一串字符/...。
  3. 只取 api.day.app/ 后、下一个 / 前的字符，填入 Device Key；服务器保持默认。
  4. 保存后收到“NJFU GPA 测试通知”即成功。参考 Bark 官方说明。

飞书、钉钉、企业微信、Slack

先建一个只负责收通知的群，再添加“自定义机器人”，最后复制 Webhook 地址。

  1. 进入群设置或群管理，找到“机器人 / 群机器人”。
  2. 添加“自定义机器人”或“Incoming Webhook”，名称可填 NJFU GPA。
  3. 复制完整的 HTTPS Webhook 地址，回到设置页粘贴到对应输入框。
  4. Webhook 相当于密码，不要发到聊天、截图或公开仓库。保存后以测试消息为准。

官方步骤：飞书 · 钉钉 · 企业微信 · Slack

Telegram

  1. 打开官方 @BotFather，发送 /newbot，按提示获得 Bot Token。
  2. 打开刚创建的机器人，先发送一条消息；机器人不能主动开始会话。
  3. 浏览器打开 https://api.telegram.org/bot你的Token/getUpdates，在返回内容中找到 chat 下的 id。
  4. 分别填写 Bot Token 和 Chat ID。两项都属于敏感信息。

参考 Telegram 官方教程。

ntfy

  1. 安装 ntfy App，或打开 ntfy 网页版。
  2. 订阅一个足够随机、别人猜不到的 Topic。
  3. 保持服务器 https://ntfy.sh，填写同一个 Topic；公开服务器未使用账号时 Token 可留空。
  4. Topic 可能充当访问凭据，不要使用姓名或学号，也不要公开。

怎样才算配置成功

  1. 回到私有设置页，只填写你选中的一种渠道。
  2. 保持“保存后触发一次通知测试”开启，再点“完成配置”。
  3. 必须在目标 App 或邮箱里实际收到“NJFU GPA 测试通知”；只看到网页提示“已启动”不算成功。
  4. 没收到时，删除该渠道旧配置后重新获取凭据，不要把凭据发给他人排查。
